## Overview

To build an advanced retriever that support query search with filtering with higher precision, lower hallucination and fully traceble context.

## Hybrid Search (Semantic + Keyword)

This combines **Keyword Search(Sparse Embeddings)** with **Semantic Search(Dense Embeddings)** to achieve higher accuracy.

Sematic Search - Captures Meaning<br>
Keyword Search - Matches Exact Words

In this system I implemented hybrid search through Loading the FAISS vector index(Semantic Search) and builded Keyword Search using BM25.

## Reranking

I used reranking to filter out the retrieved document which are slightly off topic and loosely related.

I implemented Cross Encoder Reranking here for reranking.

### Cross-Encoder Reranking

I used **cross-encoder/ms-marco-MiniLM-L-6-v2** model for Cross Encoding.

## Deduplication

I used Deduplication for removing repeated or highly similar text chunks before sending them to the LLM.

As I do hybrid retrieval there is Semantic and Keyword search and there is a high chance the same chunk appears in both lists so to avoid this deduplication is necessary.

## Metadata Filtering

I have implemented metadata filtering that helps in restricting retrieval scope to improve precision.

## Token-Aware Context Builder

Implemented Traceble Context Sources as through this we can trace back the exact chunks of source document used to generate the answer generated by the LLM.

This is necessary in the system like this to verify correctness, to detect hallucination.

I have implemented in this flow:

- Iteratively add ranked chunks
- Stop when token limit reached
- Maintain structured formatting with traceable metadata
